# bachata-reengage

Go-сервис для реактивации затухших чатов Bachata. MVP только читает данные из Bachata API, сохраняет состояние цикла напоминаний в PostgreSQL и пишет структурированные логи вместо реальной отправки сообщений.

## Что делает сервис

- опрашивает Bachata API по расписанию;
- находит диалоги без телефона, где последнее сообщение написал клиент;
- симулирует первое напоминание после 3 суток;
- симулирует второе напоминание после 7 суток;
- отменяет второе напоминание, если телефон появился между первым и вторым этапом;
- публикует Prometheus-метрики на `/metrics`.

## Конфигурация

Используются переменные окружения из [configs/.env.example](/Users/turkin/code/bachata-reengage-workspaces/TUR-13/configs/.env.example).

## Запуск

```bash
go run ./cmd/reengage
```

Сервис сразу выполняет одну итерацию, затем повторяет её с интервалом `POLL_INTERVAL`.
